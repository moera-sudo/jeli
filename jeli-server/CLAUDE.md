# Project Jeli
Проект Jeli - это веб-приложение для хранения своих генеалогических древ. На основе алгоритма мэтчинга(см. ..hackaton/docs/) серверная часть должна решать проблему отсутствия связи и информации о своих дальних родственников. При загрузке своего древа - сервер ищет генеалогические древа других людей зарегистрированных в сервисе, сопоставляет и при наличии совпадений предлагает мэтч древа и чат с твоими дальними родственниками внутри приложения. 

## Stack
Backend : Python[FastAPI], uv
Frontend : React
Database : PostgreSQL, SQLAlchemy 2.0
Infra: Docker, docker-compose

## Architecture
Проект представляет из себя клиент-серверную монолитную архитектуру, все сторонние сервисы должны автоматически запускаться в docker-compose.yml вместе с бэкендом. Фронт-енд часть в docker-compose бэкенда не входит

## Coding Standarts
Здесь описаны главные рекомендации что Claude Code должен делать и как
- Все сложные команды или многоэтапные команды должны быть записаны в Makefile для удобства разрабокти, самому Claude Code тоже рекомендуется использовать команды оттуда а не их прямое написание
- При комментировании Claude Code должен придерживаться строгого стандарта комментариев: Не использовать многострочные комментарии, все комментарии делятся на : * - для описания методов ** - важные ! - критически опасно TODO - сделать позже ? - комментарий с вопросом @param MyParam - описание параметров к методу и стандартные комментарии для языка Python. Нельзя использовать многострочные комментарии, в особенности комментарии покрытые сверху и снизу длинным тире
- FastAPI предоставляет отличный функционал для автоматической Swagger генерации. Claude Code обязан подробно описывать каждый эндпоинт и роутер в соответствии с детально описанной логикой самого роута и синтаксиса FastAPI. Абсолютная вся видимая документация в Swagger должна быть написана на русском языке
- Правило Логгирования: Все процессы в приложении должны сопроваждаться высоким уровнем покрытия логами, важно - правильно используй все доступные уровни логгирования. Все логи без исключения должны логгироваться на английском языке

## Tips
Здесь описаны второстепенные рекомендации, которые при наличии архитектурных причин можно не выполнять
- Использовать uvloop и httptools(обе зависимости уже установлены через uv)
- Использовать async for вместо while True нотацию
- Использовать HTTPX AsyncClient вместо TestClient
- Использовать Lifespan State вместо app.state
